# Article #1 — Thesis and Scope Definition

---

## Working Title

Cognitive Load Is the Missing Layer in Developer Experience

---

## Core Thesis

Modern engineering systems optimize for procedural efficiency but rarely account for individual cognitive capacity. Agile rituals, sprint cycles, and productivity metrics implicitly assume uniform working memory and context-switching tolerance. When cognitive load is ignored, developer experience becomes misaligned with human limits — especially for neurodivergent engineers — leading to burnout, performance degradation, and unnecessary friction.

This article argues that cognitive load should be treated as a first-class design constraint in engineering systems.

---

## Structural Pillars

1. **Cognitive Load Theory Basics**
   - Working memory is limited.
   - Tasks impose intrinsic and extraneous load.
   - Overload reduces performance predictably.

2. **Agile and Process Assumptions**
   - Frequent context switching is embedded by design.
   - Ritual-heavy environments add extraneous load.
   - Estimation and interruption overhead compound the cost.

3. **Individualized Developer Experience**
   - Cognitive capacity varies significantly across engineers.
   - ADHD and executive function variability are common and underacknowledged.
   - Process uniformity does not produce experiential uniformity.

4. **The Invisible Cognitive Tax**
   - Overload amplifies error rates.
   - Sustained overload accelerates burnout.
   - Overload is routinely misattributed to character or motivation.

5. **Systems-Level Reframe**
   - Extraneous load can be reduced through deliberate process design.
   - Engineering systems can be designed for cognitive sustainability.
   - Aligning process with human constraints is an engineering problem, not a wellness initiative.
